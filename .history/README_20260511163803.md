Project gồm:
- Frontend: Angular 20
- Backend: ASP.NET Core 9
- Database: SQL Server

SQL Server chỉ lưu dữ liệu mã hóa, hash và SecureIndex. SQL Server không giữ khóa mã hóa, không lưu plaintext và không thực hiện giải mã dữ liệu. Việc mã hóa, giải mã, tạo hash và kiểm chứng kết quả được xử lý ở tầng ASP.NET Core Middleware.

---

## Cách chạy project
### 1. Database

Mở SQL Server Management Studio, tạo hoặc restore database `HRM`, sau đó chạy file:

Database/script.sql

### 2. Backend
Chạy backend:
cd HRM.API
dotnet restore
dotnet run

### 3. Frontend
Chạy frontend:

cd HRM.UI
npm install
ng serve

Frontend chạy tại:

http://localhost:4200

## Public mode và Private mode

Hệ thống có hai chế độ truy cập dữ liệu: **Public mode** và **Private mode**.

### Public mode

Public mode dùng cho người dùng chỉ được xem dữ liệu không nhạy cảm.

Ở chế độ này:

- Họ tên, CMND và số tài khoản bị che bằng `****` hoặc trả về `null`.
- Không cho phép thêm nhân viên.
- Không cho phép sửa nhân viên.
- Không cho phép xóa nhân viên.

Ví dụ:

MaNV: NV001  
HoTen: ****  
CMND: ****  
Mobile: 0909123456  
Email: user@example.com

### Private mode

Private mode dùng cho người có quyền xem dữ liệu đầy đủ.

Ở chế độ này:

- Backend lấy ciphertext từ SQL Server.
- ASP.NET Core giải mã dữ liệu bằng AES.
- API trả dữ liệu rõ về Angular để hiển thị.
- Cho phép thêm, sửa, xóa nhân viên.
- Cho phép tìm kiếm theo họ tên hoặc CMND đầy đủ.

Ví dụ:

MaNV: NV001  
HoTen: Nguyễn Văn A  
CMND: 123456789  
Mobile: 0909123456  
Email: user@example.com

## Luồng tìm kiếm dữ liệu mã hóa

Web UI gửi truy vấn  
→ API tạo hash hoặc n-gram hash  
→ SQL Server lọc ứng viên trên SecureIndex/hash  
→ SQL Server trả ciphertext  
→ API giải mã tập ứng viên  
→ API kiểm chứng điều kiện gốc  
→ Web UI hiển thị kết quả rõ

Tìm kiếm theo họ tên sử dụng `SecureIndex`.

Tìm kiếm theo CMND sử dụng `CMNDHash`, nên cần nhập đầy đủ CMND.

---

## Dữ liệu lưu trong SQL Server

SQL Server chỉ lưu:

- Ciphertext: `I_Holot`, `I_Ten`, `I_CMND`, `I_Sotaikhoan`
- Hash tra cứu: `CMNDHash`, `SotaikhoanHash`
- SecureIndex: `RecordId`, `TableName`, `ColumnName`, `GramHash`, `Position`

SQL Server không lưu:

- Họ tên plaintext
- CMND plaintext
- Số tài khoản plaintext
- Plaintext n-gram
- Khóa mã hóa

## Kiểm tra nhanh

Kiểm tra dữ liệu nhân viên:

SELECT TOP 20
    Id_NV,
    Holot,
    Ten,
    CMND,
    I_Holot,
    I_Ten,
    I_CMND,
    CMNDHash
FROM dbo.Tbl_Nhanvien
ORDER BY Id_NV DESC;

Kết quả đúng sau khi migrate và xóa plaintext:

Holot = NULL  
Ten = NULL  
CMND = NULL  
I_Holot = 0x...  
I_Ten = 0x...  
I_CMND = 0x...  
CMNDHash = 0x...

Kiểm tra SecureIndex:

SELECT TOP 20 *
FROM dbo.SecureIndex
ORDER BY Id DESC;

Bảng `SecureIndex` chỉ lưu `GramHash`, không lưu `GramValue`.
